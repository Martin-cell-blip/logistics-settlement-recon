# 物流结算异常复核 Copilot：MVP PRD

> 版本：v0.1 ｜状态：作品集 MVP ｜范围：**建议与审批，不执行资金动作**

## 1. 问题与目标

结算专员在承运商账单到达后，需跨订单、签收、系统应计和合同规则定位异常，并判断拒付、追回、催票或确认成本。人工逐单查证慢，且相同异常会因证据组织和经验差异出现处置口径不一致。

本 MVP 的目标不是替代结算专员，而是把“查证—归因—建议—审批”变成可追溯的案件流：

- 将异常案件、金额影响和原始证据集中呈现；
- 用受控工作流生成**可解释建议**，不让模型计算金额或执行处置；
- 让审核人一键采纳、驳回或升级，并沉淀可用于后续评测的反馈。

## 2. 用户、场景与范围

| 角色 | 核心任务 | 典型痛点 |
|---|---|---|
| 结算专员 | 核验账单差异、提交处置建议 | 在多个表格、订单与条款间反复跳转 |
| 结算主管 | 审批高金额或低置信度案件 | 难以快速判断建议是否有完整证据 |
| 财务/风控 | 追踪风险敞口和处置口径 | 缺少统一审计轨迹 |

本项目使用 Olist 公开数据与**合成承运商账单**。未声称已完成真实企业访谈；上述角色和痛点是基于结算流程进行的产品假设，下一步应以 5–8 位一线结算人员访谈验证。

首版只覆盖 `MISSING_ORDER`、`NOT_DELIVERED`、`DUPLICATE`、`OVERBILLED`、`UNDERBILLED`、`NOT_BILLED` 六类异常；不覆盖真实支付接口、合同 OCR、跨币种税费和自动催收。

## 3. 用户故事与验收标准

1. **作为结算专员**，我可按优先级查看案件及金额影响，避免先处理低影响异常。  
   验收：队列展示优先级、状态、金额、建议动作和人工审批状态。
2. **作为结算专员**，我可打开案件，看到订单、账单、系统应计和建议依据。  
   验收：每条建议都关联结构化证据摘要，缺少证据时只能给出“人工复核”。
3. **作为结算主管**，我可采纳、驳回或升级建议，并留下备注。  
   验收：决策写入审计记录；界面和 API 均不调用支付、退款、追回或催票接口。

## 4. MVP 功能与非功能要求

| 模块 | MVP 功能 | 明确边界 |
|---|---|---|
| 异常队列 | 优先级、差异类型、金额影响、筛选 | 不代表最终会计结论 |
| 案件证据 | 订单状态、签收、SOR、账单汇总、规则依据 | 不向模型暴露无关客户信息 |
| AI 建议 | 输出裁定、建议动作、置信度、理由 | 只解释，不覆盖 SQL 金额结果 |
| 人工审批 | 采纳/驳回/升级、审核人和备注 | 所有资金类动作必须人工审批 |
| 审计记录 | case_id、策略版本、时间与决定 | 不保存模型思维链或敏感密钥 |

质量门槛：P95 页面/API 响应 < 3 秒（本地数据规模）；任何证据不足案件的“人工复核”兜底率为 100%；建议不得自动改变财务系统状态。

## 5. 迭代假设与埋点

首轮验证的北极星指标是**人工复核后被采纳的高置信度建议占比**。同时记录每案处理时长、建议采纳/驳回/升级率、证据缺失率、各异常类型的金额风险覆盖，以及 API 时延。若“高置信度但被驳回”集中在某一异常类型，优先检查证据链和规则，而非盲目更换模型。
