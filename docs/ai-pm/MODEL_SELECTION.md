# 模型选型与上线闸门

本场景不是让模型做金额推理，而是生成短、结构化、可引用证据的审核摘要。因此优先选择满足质量门槛的轻量模型，而不是默认选择最大模型。

## 1. 基线与候选

| 方案 | 角色 | 当前状态 | 选型判断 |
|---|---|---|---|
| 确定性规则 | 裁定、动作、金额和安全回退 | 已实现；21 条规则回归和 10 条安全案例 | 唯一业务决策来源 |
| 轻量结构化模型 | 日常证据摘要 | 接口与评测框架已实现，真实结果待有密钥后测量 | 达标时优先，控制时延和成本 |
| 增强结构化模型 | 困难案例挑战者 | 尚未测量 | 只有质量增益显著且成本可接受时采用 |

候选模型必须通过环境变量指定完整模型快照，避免文档中的营销名称与实际运行版本不一致。

## 2. 同集比较指标

| 指标 | 发布门槛 |
|---|---|
| Schema 合规率 | 100% |
| 不存在证据引用率 | 0% |
| 规则裁定/动作冲突放行率 | 0% |
| 过度置信放行率 | 0% |
| 关键证据覆盖率 | ≥ 95%（独立标注集） |
| P95 模型时延 | < 3 秒（作品集本地目标） |
| 单案 Token/成本 | 实测记录，不预设虚构数字 |

业务采纳率和处理时长必须另行报告，不能由离线模型分数代替。
