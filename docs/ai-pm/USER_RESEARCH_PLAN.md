# 用户研究计划与访谈记录模板

> 当前状态：尚未完成真实企业访谈。本文件是可执行计划，不是虚构研究结果。

## 1. 招募与样本

- 5–8 位有物流账单、采购对账或应付结算经验的一线人员；
- 至少 1 位审批主管或财务风控人员；
- 尽量覆盖高频小额和低频高额两类业务。

## 2. 核心问题

1. 最近一次异常账单从发现到关闭经历了哪些步骤？
2. 哪些系统、表格或合同条款需要反复切换？
3. 最耗时的是找证据、判断原因、写说明还是等审批？
4. 哪些异常可以规则化，哪些必须由人判断？
5. 什么证据缺失时绝不能给出处置建议？
6. 主管采纳或驳回建议时最关注什么？
7. 如果系统只生成证据摘要而不自动执行，是否有使用价值？
8. 如何衡量成功：处理时长、返工、差错、风险金额覆盖还是其他指标？

避免询问“你是否喜欢 AI”这类引导性问题；优先追问最近发生的真实案例和现有替代方案。

## 3. 单次访谈记录

```text
访谈编号：
角色/经验：
业务规模与异常类型：
当前流程：
使用工具：
最近真实案例：
主要痛点及证据：
风险/合规限制：
对原型的关键反应：
可验证机会：
不支持原假设的证据：
后续产品决策：
```

记录只保存岗位和流程信息；不得上传客户名称、支付账户、个人联系方式或未脱敏账单。

## 4. 综合输出

访谈完成后产出：

1. 痛点频率 × 风险金额矩阵；
2. 当前人工基线：每案处理时长、返工率和升级率；
3. 必须证据清单和禁止自动化边界；
4. 经证据支持、被推翻和仍待验证的产品假设；
5. PRD v0.3 变更记录。
